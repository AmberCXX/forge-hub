# iMessage 通道 · 信任模型说明

## 和其他通道的区别

Telegram / 微信 / 飞书的消息来自**外部网络**——陌生人主动发消息给 bot，是真实的安全事件。

iMessage 读的是**本地 chat.db**——macOS 把所有短信和 iMessage 存在这个 SQLite 数据库里。运营商通知、银行验证码、快递提醒都在里面。不在白名单的消息不是"有人试图联系你的 bot"，只是"你收到了一条你没订阅的短信"。

## 非白名单消息的处理

| 行为 | Telegram / 微信 / 飞书 | iMessage |
|------|----------------------|----------|
| 记录到 evidence vault | 是 | 是 |
| 触发安全事件 + 警报 | 是 | **否** |
| 推送给实例 | 否 | 否 |

iMessage 的非白名单消息仍然写入 evidence vault（`fh hub security evidence` 可查），但不触发 `recordSecurityEvent`，不向实例推送安全警报。

## 为什么这样设计

安全警报应该只在有安全意义的时候触发。银行短信触发"未授权访问尝试"警报，语义是错的——它会让用户误以为有人在攻击通道，实际上只是本地数据库里的正常噪音。

evidence vault 保留是因为审计记录本身有价值——事后可以查"过去一个月有哪些号码给我发过消息"。
